# 色彩规则

基于《数据之美》的色彩使用原则，以及 ColorBrewer、Viridis 等学术色彩方案的实践。

## 核心原则

### 1. 色板与数据类型匹配

| 数据类型 | 适用色板         | 示例场景                    |
|----------|------------------|----------------------------|
| 分类     | categorical      | 产品类别、地区、部门         |
| 顺序     | sequential       | 温度、密度、收入等级         |
| 发散     | diverging        | 同比变化率、偏差值、净推荐值  |
| 高亮     | highlight        | 目标值、预警线、基准线       |

### 2. 色盲友好

约 8% 的男性和 0.5% 的女性有色觉障碍。默认色板应满足：
- 红绿盲（Protanopia/Deuteranopia）：避免仅靠红-绿区分
- 蓝黄盲（Tritanopia）：较少见，但高对比度的蓝-黄也应避免

默认 categorical 色板经过 ColorBrewer 验证，10 种颜色均可被色盲人群区分。

### 3. 语义一致性

- 正增长/好 → 蓝或绿色系
- 负增长/差 → 红或橙色系
- 中性/参考 → 灰色系
- 高亮/警告 → 红色系（仅用于需要引起注意的信息）

### 4. 颜色数量控制

- 分类色板建议 ≤ 7 种颜色（超过时考虑合并小类为"其他"）
- 顺序色板建议 ≥ 5 个梯度（太少无法区分）
- 一张图表中高度饱和的颜色不超过 3 种

## 自然语言颜色指令映射

当调用者提出颜色相关的自然语言要求时，按以下映射调整：

| 自然语言                     | 美学参数调整                                      |
|-----------------------------|--------------------------------------------------|
| "颜色更活泼/鲜艳"             | 色板饱和度 ×1.3，使用 multi_hue 替代 single_hue     |
| "颜色素雅/低调"               | 饱和度 ×0.7，亮度 +10%                             |
| "深色主题/暗色背景"           | 背景 #1a1a1a，文字 #f0f0f0，色板亮度整体 +15%       |
| "使用公司品牌色"              | 替换 categorical[0] 为首要品牌色，其余用品牌色衍生   |
| "暖色调"                     | 限制色相范围在 0-60°（红-橙-黄）                    |
| "冷色调"                     | 限制色相范围在 180-270°（青-蓝-紫）                 |
| "对比度更强"                  | 色板相邻颜色色相间距至少 30°                        |
| "柔和对比较弱"                | 色板颜色色相间距 ≤ 15°，整体降低饱和度               |
| "突出 {类别}"                | 目标类别使用 highlight.primary，其余类别 opacity: 0.3 |
