# 设计一致性规则

当一次输出包含多个图表时，以下规则确保视觉统一。

---

## 规则 1：排版层级统一

同一输出中的所有图表，相同角色的文本使用相同的排版规格。

```
检查清单：
□ 所有图表的主标题字号是否一致？（h1: 16px/600）
□ 所有子标题字号是否一致？（h2: 13px/400）
□ 所有轴标题字号是否一致？（h3: 11px/400）
□ 所有刻度标签字号是否一致？（h4: 10px/400）
□ 所有图例文本字号是否一致？（h5: 10px/400）
□ 所有脚注字号是否一致？（h6: 9px/400）
```

## 规则 2：颜色语义统一

同一数据实体在所有图表中使用相同颜色。

```
示例：
如果"华东区"在图表 A 中是 #4269d0（蓝色），
那么它在图表 B、C 中也必须是 #4269d0。
即使图表类型不同（柱状图 vs 折线图 vs 饼图），颜色映射保持一致。
```

**颜色映射表**：当生成多图表时，首先建立一个 `entity → color` 映射表，所有图表引用该表。

## 规则 3：间距统一

同一画布/页面上的多个图表，使用统一的边距和间距。

```
检查清单：
□ 所有图表的 margin 是否一致？
  (top: 40, right: 30, bottom: 50, left: 60)
□ 图表之间的间距是否一致？（24px）
□ 相同宽高比？（默认 1.6:1）
```

## 规则 4：线型与点型统一

同类型的视觉元素使用相同的样式。

```
检查清单：
□ 同数据系列的折线宽度是否一致？（2px）
□ 同数据系列的点大小是否一致？（4px 直径）
□ 网格线样式是否一致？（0.5px, #E0E0E0, dash [2,2]）
□ 参考线样式是否一致？（1px, #ff725c, dash [4,4]）
□ 趋势线样式是否一致？（1.5px, #ff725c, dash [6,3], opacity 0.8）
```

## 规则 5：数值格式统一

同一类数值在所有图表中使用相同格式。

```
检查清单：
□ 小数位数是否一致？（默认 1 位）
□ 百分比格式是否一致？
□ 日期格式是否一致？
□ 单位标注是否一致？（万、亿、%、‰）
□ 千分位分隔符是否一致？
```

## 规则 6：标注风格统一

数据标签、注释、高亮的样式在所有图表中保持一致。

```
检查清单：
□ 数据标签的字体大小和颜色是否一致？
□ 标注线的样式是否一致？
□ 高亮区域的样式是否一致？
□ 提示框（Tooltip）的样式是否一致？
```

---

## 一致性冲突处理优先级

当用户定制与默认规范冲突时：

1. **用户明确指定** → 最高优先级，覆盖默认规范
2. **图表类型特殊要求** → 如饼图不需要轴标签，可省略相关规则
3. **默认规范** → 上述两者未覆盖时，严格遵循默认规范

## 一致性校验工具

执行 `scripts/theme_validator.py` 可检查输出的 JSON 配置是否符合规范。
