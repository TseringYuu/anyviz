# 图例布局规则

图例是图表的信息入口，但也是最容易出布局问题的区域。本文档汇总图例 positioning 的硬性规则，适用于所有图表类型。

---

## 规则 1：图例必须在 viewBox 边界内

**问题**：多系列图表的图例使用 `transform="translate(x,y)"` 定位时，最后一项文字可能超出 SVG viewBox 宽度，导致渲染时被裁剪。

**规则**：图例最右侧文本的结束 x 坐标必须 ≤ viewBox 宽度 - 10px 安全边距。

**验证方法**（生成图例后必查）：

```
图例总宽度 = Σ(每项宽度)
每项宽度 = 线段长度(22px) + 间距(8px) + 文本宽度(估算: 字数 × 字号 × 0.6) + 项间距(20px)

图例起始 x = (viewBox_width - 图例总宽度) / 2   ← 居中
            或 viewBox_left_margin             ← 左对齐

约束：图例起始 x + 图例总宽度 ≤ viewBox_width - 10
```

**示例**（viewBox 宽度 680px）：

```
3 项图例，每项约 180px → 总宽 540px
居中起始 x = (680 - 540) / 2 = 70  ← 正确
左对齐起始 x = 50                   ← 正确
左对齐起始 x = 110                  ← 错误：110 + 540 = 650，末项文字延伸至 ~690，超出 viewBox
```

## 规则 2：图例不得与数据标签、轴标签重叠

**问题**：雷达图底部维度标签与图例在同一 y 区域，导致文字重叠。

**规则**：图例区域的 y 范围必须与所有轴标签/数据标签的 y 范围保持 ≥ 15px 间距。

**常见冲突场景**：

| 图表类型 | 冲突点 | 解决方案 |
|----------|--------|----------|
| 雷达图 | 底部维度标签 vs 图例 | 图例 y 起始 ≥ 底部维度标签 y + 22px |
| 柱状图 | 负值柱下方标签 vs 类别标签 | 类别标签 y ≥ 负值标签 y + 15px |
| 横向条形图 | 负值标签 vs 类别标签 | 类别标签与值标签分置两侧（类别左对齐左边缘，负值右对齐柱左端） |
| 折线图 | 末点数据标签 vs 图例 | 图例置于底部，数据标签置于线条末端上方 |

## 规则 3：多系列图例使用 2×N 网格而非单行

**问题**：4 个以上图例项单行排列时，总宽度容易超出 viewBox。

**规则**：图例项 ≥ 4 时，改用 2 行 × ceil(N/2) 列布局，行间距 22px。

**布局计算**：

```
项数 N
列数 C = ceil(N / 2)
行数 R = 2

每列宽度 = viewBox_width / C × 0.8   ← 留 20% 边距
行间距 = 22px

第 i 项的位置：
  row = i // C
  col = i % C
  x = margin_left + col × column_width
  y = legend_top + row × 22
```

## 规则 4：图例默认水平居中

**规则**：图例整体应在画布水平方向居中，除非有数据元素阻挡。

**居中计算**：

```javascript
const legendWidth = items.reduce((sum, item, i) => {
  return sum + item.markerWidth + item.textWidth + (i < items.length - 1 ? itemGap : 0);
}, 0);

const legendX = (viewBoxWidth - legendWidth) / 2;
```

**例外**：当图表底部有脚注（数据来源等）时，图例与脚注之间需 ≥ 15px 间距，此时可能需要缩小图例字号或调整布局。

## 规则 5：学术主题下的图例线型冗余

**问题**：academic 主题使用灰度色板（#333333 / #666666 / #999999 / #BBBBBB），仅靠颜色难以区分 3 个以上系列。

**规则**：使用 academic 主题时，图例必须同时标注线型差异，数据线也必须使用对应线型。

**线型映射**（按系列顺序）：

| 系列序号 | 颜色 | 线型 | stroke-dasharray |
|----------|------|------|------------------|
| 1 | #333333 | 实线 | 无 |
| 2 | #666666 | 长虚线 | "5,3" |
| 3 | #999999 | 短虚线 | "3,2" |
| 4 | #BBBBBB | 点线 | "1,2" |
| 5+ | 循环上述 + 降低 opacity | — | — |

**图例标记**：每项图例的线段必须使用与数据线相同的 `stroke-dasharray`，不能仅用颜色区分。
