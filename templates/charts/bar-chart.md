# 柱状图 / 条形图（Bar Chart）

## 适用场景
- 比较不同类别的数值大小
- 展示排名
- 最佳类别数：2-15 个

## 数据格式
```json
{
  "categories": ["华东", "华南", "华北", "西南", "西北"],
  "series": [
    { "name": "Q1", "values": [320, 280, 250, 180, 120] },
    { "name": "Q2", "values": [350, 290, 270, 195, 130] }
  ]
}
```

## 美学参数

```json
{
  "bar": {
    "width_ratio": 0.7,
    "corner_radius": 0,
    "min_height": 2,
    "label_position": "outside",
    "label_offset": 4
  }
}
```

## 设计要点

1. **排序**：类别应按数值排序（降序或升序），除非有自然顺序（如月份、年龄段）
2. **方向选择**：
   - 类别标签较短（< 6 字）→ 纵向柱状图
   - 类别标签较长或类别较多 → 横向条形图
3. **Y 轴基线**：柱状图必须从 0 开始（否则柱高比例会误导读者）
4. **颜色**：单系列用 #4269d0；多系列按 categorical 色板顺序
5. **标签位置**：正值标签在柱上方，负值标签在柱下方
6. **负值柱的类别标签避让**：当存在负值柱时，X 轴类别标签会被负值柱下方的数据标签挤压。类别标签 y 坐标 ≥ max(负值标签 y) + 15px。若负值较大（柱高 > 30px），类别标签与 0% 基线的距离应 ≥ 负值柱高度 + 负值标签高度(14px) + 间距(15px)
7. **横向条形图的标签分侧**：横向条形图有正负值时，类别标签左对齐于画布左边缘（x=15），正值标签右对齐于柱右端外侧，负值标签右对齐于柱左端外侧。三者不可出现在同一 x 区域

## 变体

### 分组柱状图（Grouped）
多系列并排，用于比较不同系列在同一类别上的差异

### 堆叠柱状图（Stacked）
多系列叠加，用于展示总量及各部分贡献（最多 5 个系列）

### 百分比堆叠柱状图（100% Stacked）
归一化到 100%，用于展示各部分的相对比例

### 棒棒糖图（Lollipop）
柱状图的简化版，用线和点替代柱体，减少"墨水"使用

### 双向柱状图（Diverging Bar）
数据有正负值时，向两侧延伸，中间对齐在零点

**双向柱状图的标签布局**（高频出错点）：

纵向双向柱状图：
- 正值标签：柱顶上方 7px，text-anchor="middle"
- 负值标签：柱底下方 14px，text-anchor="middle"
- 类别标签：0% 基线下方 max(负值柱高度, 20px) + 25px，确保不被负值标签遮挡

横向双向条形图：
- 类别标签：text-anchor="start"，x=15（画布左边缘），与 0% 基线无关
- 正值标签：柱右端 + 6px，text-anchor="start"
- 负值标签：柱左端 - 6px，text-anchor="end"
- 类别标签与负值标签在 x 轴方向天然分离，不会重叠

```
横向布局示意：
[类别标签]        0% |███████ 正值  +15.7%
                    |    负值 -2.7% |████
```

**常见错误**：将类别标签 right-aligned 在 0% 基线左侧（如 x=190 text-anchor="end"），导致与负值标签（right-aligned at bar_left-6）在同一 x 区域重叠。正确做法是类别标签 left-aligned 在画布最左侧。
